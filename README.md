# 智慧园林 AI 平台（Smart Garden AI）

> 面向市政园林管理的 AI 综合解决方案 | 知识库 + AI 识别 + 无人机巡检 + 数字孪生

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

---

## 一、项目定位

为城市园林局、公园管理中心、绿化养护单位提供一套覆盖**植保知识 → AI 视觉识别 → 无人机巡检调度 → GIS 看板 → 养护决策**的完整智慧园林 SaaS 平台。

| 角色 | 核心需求 |
|------|------|
| 园林局管理员 | 掌握辖区植被资产底数、病虫害趋势、养护成本 |
| 巡检人员/无人机飞手 | 接收任务、执行航拍、上传图片 |
| AI 算法工程师 | 获取标注数据、评估模型指标、迭代模型 |
| 养护工人 | 接收工单、现场处置、反馈结果 |

## 三、模块设计

### 模块一：智慧园林知识库

**定位：** 面向园林行业的专用 RAG 知识库，覆盖苗木、病虫害、养护标准三大领域。

**数据来源：**
- 公开资料：园林苗木图鉴、植物病虫害图谱、市政园林养护标准（国标/地方标准）
- 历史积累：过往巡检记录、病虫害案例库、养护方案库
- 用户上传：园林局内部养护手册、专家经验文档

**核心功能：**

| 功能 | 描述 |
|------|------|
| 苗木百科 | 按树种查询形态特征、生长习性、适宜环境、常见病虫害。每树种关联养护方案 |
| 病虫害图鉴 | 按虫害名称查询识别特征（高清图+文字描述）、危害症状、防治方法、推荐药剂 |
| 养护标准库 | 按苗木类型查询修剪周期、施肥方案、浇水频次、冬季防护要求 |
| AI 问答 | 基于 RAG 的自然语言查询："桂花树叶发黄是什么原因？如何处理？"→返回症状分析+处理建议+关联养护标准 |
| 知识更新 | 园林局管理员可上传内部资料补充知识库，支持 PDF/Word/图片多种格式 |

**验收标准：** 知识库覆盖深圳常见园林苗木 ≥100 种、病虫害 ≥50 种、养护标准 ≥30 项。AI 问答 Top-3 召回率 ≥85%。

---

### 模块二：AI 视觉识别

**定位：** 挂载于无人机/地面相机，自动识别植被病虫害、长势异常、设施破损。

**分层策略：**

| 层次 | 部署位置 | 能力 | 适用场景 |
|------|------|------|------|
| L1 边缘轻量 | 无人机机载/边缘盒子 | 简单二分类（正常/异常）、疑似区域框选 | 实时航拍初筛，减少云端传输量 |
| L2 云端精细 | GPU 集群 | 病虫害细粒度分类、严重程度评估、面积估算 | 航拍后批量推理，生成详细报告 |
| L3 多模态融合 | 云端 | 结合可见光+多光谱+热红外 | 高价值区域精细巡检（名木古树） |

**识别类目与指标：**

| 识别任务 | 召回率指标 | 误报率上限 | 优先级 |
|------|:--:|:--:|:--:|
| 植被枯死/枯黄检测 | ≥90% | ≤10% | P0 |
| 常见虫害识别（蚜虫、介壳虫、天牛等8类） | ≥85% | ≤15% | P0 |
| 常见病害识别（白粉病、锈病、炭疽病等6类） | ≥80% | ≤20% | P1 |
| 草坪斑秃/退化检测 | ≥85% | ≤10% | P1 |
| 乔木倾斜/倒伏检测 | ≥95% | ≤5% | P0 |
| 灌溉设施破损检测 | ≥80% | ≤15% | P2 |
| 外来入侵物种识别 | ≥75% | ≤25% | P2 |
| 名木古树长势评估 | ≥85% | ≤10% | P1 |

**验收机制：** 每项识别任务独立评估。算法团队交付模型前需提供测试集上的指标报告。产品经理组织园林局专家进行人工抽检（随机100张图片），对比人工标注和AI识别结果，达标方可上线。

---

### 模块三：无人机巡检调度

**定位：** 管理无人机巡检任务的全生命周期——从航线规划到航拍图片入库。

| 功能 | 描述 |
|------|------|
| 航线规划 | 在 GIS 地图上绘制巡检区域，自动生成航点航线。支持导入 KML/GeoJSON |
| 任务创建 | 选择航线、设定飞行参数（高度/速度/重叠率）、指派飞手 |
| 实时图传 | 飞手通过小程序查看无人机实时画面（需无人机支持） |
| 航拍入库 | 无人机降落连接WiFi后自动上传航拍图片至云端，记录 GPS 坐标和拍摄时间 |
| 飞行记录 | 每架次记录起飞时间、降落时间、飞行距离、覆盖面积、电池消耗 |

---

### 模块四：工单与养护管理

**定位：** AI 识别出问题后，自动生成养护工单，派发给对应养护工人，闭环追踪。

| 功能 | 描述 |
|------|------|
| 智能派单 | 根据病虫害类型、地理位置自动匹配养护工人技能和负责区域 |
| 工单状态流转 | 待派发→已派发→处置中→待验收→已验收，每步记录时间和操作人 |
| 处置指引 | 工单附带 AI 推荐处置方案（药剂配方、施工方法、安全注意事项） |
| 现场反馈 | 养护工人小程序上传处置完成照片，填写实际用药量和施工情况 |
| 验收闭环 | 管理员对比处置前后图片，确认效果后关闭工单 |

---

### 模块五：GIS 可视化与数据分析

**定位：** 园林局管理层的"驾驶舱"，一张图看清辖区园林资产和养护态势。

| 功能 | 描述 |
|------|------|
| 植被一张图 | 在 GIS 地图上标注每棵乔木/灌木的位置、树种、树龄、最近养护时间 |
| 巡检轨迹回放 | 回放无人机历史飞行轨迹，叠加航拍缩略图 |
| 病虫害热力图 | 按时间区间展示病虫害发生热力分布，辅助趋势研判 |
| 养护成本统计 | 按区域、树种、养护类型统计月度/年度养护费用 |
| 名木古树专项 | 重点树木的独立电子档案，包含历年巡检图片对比、养护记录、健康评分 |

---

### 模块六：数字孪生与智能决策（长期）

| 阶段 | 内容 |
|------|------|
| 短期（0-6月） | 云端弹性 GPU 调度：航拍图片批量推理时按需启动 GPU，推理完成自动释放，降低客户运维成本 |
| 中期（6-12月） | 机载轻量 AI：植被异常检测在边缘计算盒子本地执行，只将疑似异常的图片回传云端，减少 80% 图片传输量 |
| 长期（12-24月） | 园林数字孪生：融合无人机巡检 + 土壤传感器 + 气象数据 + 历史养护记录，建立植被生长模型，为园林局提供"何时修剪/何时施肥/何时喷药"的智能养护决策建议 |

## 四、需求分层策略

| 层 | 类型 | 说明 | 示例 |
|------|------|------|------|
| 标准模块 | 所有客户通用 | 一次开发，配置化交付 | 无人机调度、工单管理、报表统计、GIS 底图 |
| 可配置模块 | 多数客户需要但参数不同 | 配置开关+参数调整 | AI 识别类目（不同城市病虫害不同）、告警阈值 |
| 定制模块 | 单个客户专属 | 单独报价，独立开发 | 专属植被识别模型训练、园区自有 GIS 图层导入 |

## 六、技术选型建议

| 层 | 推荐方案 |
|------|------|
| 前端 Web | Vue.js + Element Plus + Cesium.js（GIS） |
| 小程序 | 微信原生小程序 |
| 后端 | Java Spring Boot / Python FastAPI（AI 推理服务） |
| 数据库 | PostgreSQL（业务数据）+ PostGIS（空间数据）+ Redis（缓存/队列） |
| 知识库 | 向量数据库（Milvus/Qdrant）+ RAG 框架 |
| AI 推理 | PyTorch + ONNX Runtime（边缘）+ GPU 集群（云端） |
| 边缘计算 | NVIDIA Jetson / 瑞芯微 RK3588 |
| MQ 消息 | RabbitMQ / EMQX |
| GPU 调度 | K8s + GPU Operator（云端弹性扩缩容） |
